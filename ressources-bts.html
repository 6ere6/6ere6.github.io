SIEM IDS/IPS

CENTRALISATION DES LOGS AVEC GRAYLOG

1. Installation des conteneurs Docker

Objectif : Déployer Graylog, MongoDB et Elasticsearch pour une gestion centralisée des logs.

version: '3'
services:
  mongodb:
    image: mongo:4.2
    container_name: mongodb
    volumes:
      - mongodb_data:/data/db
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.10.1
    container_name: elasticsearch
    environment:
      - discovery.type=single-node
  graylog:
    image: graylog/graylog:4.0
    container_name: graylog
    depends_on:
      - mongodb
      - elasticsearch
    ports:
      - "9000:9000"
      - "1514:1514/udp"

Étapes suivantes : Configurer Snort pour envoyer les alertes à Graylog.

INSTALLATION DE SNORT SUR UBUNTU

1. Préparation du système

Objectif : Installer Snort pour la détection d'intrusions réseau.

# Mise à jour du système
sudo apt update && sudo apt upgrade -y
# Installation de Snort
sudo apt install snort -y
# Configuration initiale
sudo snort -T -c /etc/snort/snort.conf

Test : Lancer une attaque simulée depuis Kali Linux pour vérifier les alertes.

MONITORING RÉSEAU

INSTALLATION DE NAGIOS CORE SOUS DOCKER

1. Préparation de l'environnement

Ouvrir PowerShell sous Windows et créer un répertoire de travail :

mkdir nagios-docker
cd nagios-docker

2. Création du fichier docker-compose.yml

Créer le fichier docker-compose.yml avec la configuration suivante :

version: '3.8'
services:
  nagios:
    image: jasonrivers/nagios:latest
    container_name: nagios
    restart: always
    ports:
      - "8080:80"
    environment:
      - NAGIOSADMIN_USER=admin
      - NAGIOSADMIN_PASSWORD=admin
    volumes:
      - ./nagios/etc:/opt/nagios/etc
      - ./nagios/var:/opt/nagios/var
      - ./nagios/custom-plugins:/opt/Custom-Nagios-Plugins

Fonction : Ce fichier définit le service Nagios, les ports exposés, les identifiants d'accès et les volumes pour stocker la configuration.

3. Déploiement de Nagios

Lancer la création du conteneur avec :

docker compose up -d

Vérifier le bon démarrage avec :

docker ps

Le conteneur nagios doit apparaître dans la liste des services actifs.

4. Accès à l'interface Web

Ouvrir un navigateur et saisir : http://localhost:8080

Identifiants par défaut :

  • Utilisateur : admin
  • Mot de passe : admin

5. Configuration personnalisée

Créer un répertoire pour les configurations personnalisées :

mkdir nagios/etc/opencr_conf

Créer le fichier nagios/etc/opencr_conf/nagios-server.cfg avec le contenu suivant :

define command {
    command_name check-ping-localhost
    command_line $USER1$/check_ping -H localhost -w 40,40% -c 60,60%
}
define command {
    command_name check-ssh-localhost
    command_line $USER1$/check_ssh localhost
}
define host {
    host_name Nagios Server
    address localhost
    check_command check-ping-localhost
    max_check_attempts 3
}
define service {
    service_description SSH sur Nagios Server
    host_name Nagios Server
    check_command check-ssh-localhost
    max_check_attempts 3
}

6. Inclusion de la configuration

Éditer nagios/etc/nagios.cfg et ajouter :

cfg_dir=/opt/nagios/etc/opencr_conf

Vérifier la configuration avec :

docker exec -it nagios nagios -v /opt/nagios/etc/nagios.cfg

7. Redémarrage et validation

Redémarrer le conteneur :

docker restart nagios

Vérifier dans l'interface Web (http://localhost:8080) :

  • Menu Hosts : "Nagios Server" doit apparaître.
  • Menu Services : "SSH sur Nagios Server" doit être visible.

Permissions admin : Éditer nagios/etc/cgi.cfg et ajouter :

authorized_for_system_information=admin
authorized_for_configuration_information=admin
authorized_for_system_commands=admin
authorized_for_all_services=admin
authorized_for_all_hosts=admin
authorized_for_all_service_commands=admin
authorized_for_all_host_commands=admin

RADIUS NPS RÉSEAU

INSTALLATION ET CONFIGURATION RADIUS-NPS

1. Situation de départ

Environnement initial :

  • Serveur Windows 2019 avec Active Directory (IP : 172.20.90.100).
  • Domaine AD : mars.local.
  • Groupes AD : "escrime" et "handball".
  • Utilisateurs : "valerie" (escrime) et "nicolas" (handball).

Objectif : Configurer un serveur RADIUS-NPS pour une authentification 802.1x avec placement dynamique dans des VLANs.

2. Installation des Services de Certificats AD

Ajouter le rôle Services de certificats Active Directory :

Configurer une autorité de certification d'entreprise racine :

  • Créer une nouvelle clé privée.
  • Utiliser les paramètres de chiffrement par défaut.
  • Nommer l'autorité avec le nom de domaine (ex : mars-MARS-CA).

3. Installation du Serveur NPS

Ajouter le rôle Services de stratégie et d'accès réseau :

Sélectionner uniquement le Service NPS.

Inscrire NPS dans Active Directory pour lui permettre d'interroger la base des utilisateurs.

4. Configuration du Client RADIUS

Déclarer le client RADIUS (ex : commutateur Cisco SF300) :

  • Nom convivial : Client-Cisco-SF300
  • Adresse IP : 172.20.20.41
  • Secret partagé : Sesame9999 (à configurer aussi sur le commutateur).

5. Stratégie de Demande de Connexion

Créer une stratégie pour les connexions Ethernet 802.1x :

  • Nom : Connexion-cablée.
  • Type de port NAS : Ethernet.
  • Authentification locale (pas de transfert).

6. Stratégie Réseau (Accès au Réseau) - Groupe 'escrime'

Créer une stratégie pour le groupe "escrime" :

  • Condition : Membre du groupe AD "escrime".
  • Authentification : PEAP/MSCHAPv2.
  • Attributs RADIUS :
    • Tunnel-Type : VLAN.
    • Tunnel-Medium-Type : 802 (includes...).
    • Tunnel-Pvt-Group-ID : 2 (VLAN 2).

7. Stratégie Réseau - Groupe 'handball'

Créer une stratégie pour le groupe "handball" (similaire) :

  • Condition : Membre du groupe AD "handball".
  • Attributs RADIUS :
    • Tunnel-Type : VLAN.
    • Tunnel-Medium-Type : 802 (includes...).
    • Tunnel-Pvt-Group-ID : 3 (VLAN 3).

Rappel : L'ordre des stratégies réseau est crucial. Les stratégies spécifiques doivent précéder la stratégie "catch-all" (ex: Refus pour les autres).
